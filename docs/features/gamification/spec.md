# ゲーミフィケーション

> ステータス: **計画中（Planned）**
> 目標リリース: **未定（Future）**

---

## 概要

コープネットにポイントシステムとゲーム要素を導入し、社員の投稿・交流を促進する。いいねやコメントを受けることでポイントが貯まり、リーダーボードでランキングが表示される。将来的にはポイントを使ったミニゲームも追加する。

---

## ユーザーストーリー

### US-1: ポイントを獲得する
**ユーザーとして**、投稿やコメントの反応に応じてポイントを獲得したい。
**それにより**、投稿のモチベーションが上がる。

**受入基準:**
- [ ] いいねを受けるとポイントが加算される
- [ ] コメントを受けるとポイントが加算される
- [ ] ポイント獲得時に通知が表示される
- [ ] マイページで累計ポイントが確認できる

### US-2: リーダーボードを見る
**ユーザーとして**、ポイントランキングを確認したい。
**それにより**、他の社員の活動状況がわかり、交流が活発になる。

**受入基準:**
- [ ] 週間・月間・累計のランキングが表示される
- [ ] 上位10名のユーザー名・ポイントが表示される
- [ ] 自分の順位がハイライトされる
- [ ] 部署別ランキングも表示できる

### US-3: ミニゲームで遊ぶ
**ユーザーとして**、貯めたポイントを使ってミニゲームで遊びたい。
**それにより**、休憩時間の息抜きになり、社内コミュニケーションの話題にもなる。

**受入基準:**
- [ ] ポイントを消費してミニゲームに参加できる
- [ ] ゲーム結果に応じてボーナスポイントを獲得できる
- [ ] ゲーム履歴がマイページで確認できる

## ポイントシステム設計

### ポイント獲得ルール

| アクション | 獲得ポイント | 上限 | 備考 |
|------------|-------------|------|------|
| 投稿にいいねを受ける | +2 | 1投稿あたり最大50pt | 自分自身のいいねは対象外 |
| 投稿にコメントを受ける | +3 | 1投稿あたり最大30pt | 自分自身のコメントは対象外 |
| 毎日投稿する | +5 | 1日1回 | 連続投稿ボーナスあり |
| 連続投稿ボーナス | +2/日 | 7日連続で+14ボーナス | 途切れるとリセット |
| アンケートに回答する | +3 | 1アンケート1回 | |

### ポイント消費

| 用途 | 消費ポイント | 説明 |
|------|-------------|------|
| ミニゲーム参加 | -10 | 1回の参加費 |
| プロフィール装飾 | -50 | 特別なフレームなど（将来） |

## リーダーボード設計

### ランキング種別

| 種別 | 集計期間 | 更新頻度 |
|------|----------|----------|
| 週間ランキング | 月曜〜日曜 | 1時間ごと |
| 月間ランキング | 月初〜月末 | 1時間ごと |
| 累計ランキング | 全期間 | 1時間ごと |
| 部署別ランキング | 月間 | 1時間ごと |

### キャッシュ戦略

- ランキングデータはRedisのSorted Setで管理
- キー: `leaderboard:{type}:{period}` (例: `leaderboard:individual:weekly`)
- 1時間ごとにバッチ処理で集計・更新

## データベース

### user_points テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UNIQUEIDENTIFIER | 主キー |
| user_id | UNIQUEIDENTIFIER | FK → users |
| total_points | INT | 累計ポイント |
| available_points | INT | 使用可能ポイント |
| consecutive_days | INT | 連続投稿日数 |
| last_post_date | DATE | 最終投稿日 |
| updated_at | DATETIME2(7) | 更新日時 |

### point_transactions テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UNIQUEIDENTIFIER | 主キー |
| user_id | UNIQUEIDENTIFIER | FK → users |
| points | INT | ポイント数（+/-） |
| type | NVARCHAR(50) | 種別（like_received, comment_received, daily_post, game, etc.） |
| reference_id | UNIQUEIDENTIFIER | 関連エンティティID（投稿ID等） |
| description | NVARCHAR(200) | 説明 |
| created_at | DATETIME2(7) | 発生日時 |

### leaderboard_snapshots テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UNIQUEIDENTIFIER | 主キー |
| user_id | UNIQUEIDENTIFIER | FK → users |
| period_type | NVARCHAR(20) | weekly / monthly / all_time |
| period_key | NVARCHAR(20) | 例: 2026-W12, 2026-03 |
| rank | INT | 順位 |
| points | INT | 期間内ポイント |
| created_at | DATETIME2(7) | スナップショット日時 |

## 技術的注意事項

- ポイント計算はイベント駆動で非同期処理（いいね・コメント発生時にイベント発行）
- 二重加算防止のため、`point_transactions` で冪等性を保証する（reference_id + type のユニーク制約）
- リーダーボードはRedis Sorted Setで高速に処理し、DBへの負荷を抑える
- ポイント関連の処理はトランザクションで `user_points` と `point_transactions` を同時更新する
