# AI 機能

> ステータス: **計画中（Planned）**
> 目標リリース: **未定（Future）**

---

## 概要

コープネットにAI機能を導入し、投稿の質の向上とユーザー体験の改善を図る。コンテンツ提案、長文投稿の自動要約、社内情報検索チャットボット（RAG）、自動翻訳の4つの機能を段階的に実装する。

---

## 機能一覧

| # | 機能 | 優先度 | 説明 |
|---|------|--------|------|
| 1 | コンテンツ提案 | 高 | 投稿文のリライト・改善提案 |
| 2 | 長文自動要約 | 高 | 長い投稿の3行サマリー自動生成 |
| 3 | 社内チャットボット（RAG） | 中 | 過去の投稿・お知らせを基にした質問応答 |
| 4 | 自動翻訳 | 低 | 日本語以外の言語への翻訳（将来的な多言語対応） |

---

## 1. コンテンツ提案

### ユーザーストーリー

**ユーザーとして**、投稿を書いた後にAIから文章の改善提案を受けたい。
**それにより**、より読みやすく魅力的な投稿ができる。

### 機能詳細

| 項目 | 内容 |
|------|------|
| トリガー | 投稿作成画面で「AIに相談」ボタンをクリック |
| 入力 | ユーザーが入力した投稿文 |
| 出力 | リライト案（最大3パターン） |
| LLMモデル | Azure OpenAI GPT-4o / ローカルLLM |

### 提案パターン

| パターン | 説明 |
|----------|------|
| より丁寧な表現 | ビジネス向けの丁寧な文章に変換 |
| より簡潔に | 冗長な表現を簡潔にまとめる |
| より親しみやすく | カジュアルで読みやすい文章に変換 |

### UIフロー

1. ユーザーが投稿文を入力
2. 「AIに相談」ボタンをクリック
3. ローディング表示（2〜5秒）
4. 3つのリライト案がモーダルで表示
5. ユーザーが案を選択するか、元の文を維持
6. 選択した文が投稿フォームに反映

## 2. 長文投稿の自動要約

### ユーザーストーリー

**ユーザーとして**、長い投稿に自動で要約が付いていてほしい。
**それにより**、タイムラインをスクロールする際に内容をすばやく把握できる。

### 機能詳細

| 項目 | 内容 |
|------|------|
| トリガー | 投稿が保存された時点で非同期実行 |
| 条件 | 投稿本文が200文字以上の場合のみ |
| 出力 | 3行以内の要約文（日本語） |
| 表示場所 | タイムラインのカード上部に薄いグレーで表示 |

### 処理フロー

1. 投稿が作成される
2. バックグラウンドジョブ（Redis Bull Queue）でLLMに要約リクエスト
3. 要約が生成されたら `posts.ai_summary` カラムに保存
4. フロントエンドは次回の表示時に要約を表示

### エッジケース

| ケース | 対応 |
|--------|------|
| LLMがダウン | 要約なしで表示（機能は非必須） |
| 要約生成に時間がかかる | 最大10秒でタイムアウト、リトライ1回 |
| 不適切な要約が生成された | 管理者が要約を削除できるようにする |
| 投稿が編集された | 再度要約を生成する |

## 3. 社内チャットボット（RAG）

### ユーザーストーリー

**ユーザーとして**、社内の過去の投稿やお知らせについて質問できるチャットボットが欲しい。
**それにより**、必要な情報をすばやく見つけられる。

### アーキテクチャ

```
ユーザーの質問
    │
    ▼
┌─────────────────┐
│ Embedding Model │ ← 質問をベクトル化
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Vector Store   │ ← 類似度検索で関連投稿を取得
│  (pgvector /    │
│   Redis Search) │
└────────┬────────┘
         │ 上位5件の関連コンテンツ
         ▼
┌─────────────────┐
│    LLM (GPT)    │ ← 関連コンテンツ + 質問で回答生成
└────────┬────────┘
         │
         ▼
    回答テキスト + 参照元リンク
```

### 機能詳細

| 項目 | 内容 |
|------|------|
| 対象データ | 投稿（body）、お知らせ（title + body）、コメント |
| Embeddingモデル | Azure OpenAI text-embedding-3-small / ローカル |
| Vector Store | pgvector（SQL Server拡張）または Redis Search |
| チャンクサイズ | 500文字、オーバーラップ100文字 |
| 検索上位件数 | 5件 |

### データ更新フロー

1. 投稿・お知らせが作成/更新されるたびに Embedding を生成
2. Vector Store に保存
3. 論理削除された投稿のベクトルも削除

### UIフロー

1. サイドバーまたはフローティングボタンでチャットを開く
2. 質問を入力して送信
3. AIが回答を生成（参照元の投稿リンク付き）
4. 過去の会話履歴は保持しない（ステートレス）

## 4. 自動翻訳

### ユーザーストーリー

**ユーザーとして**、日本語以外の言語でも投稿の内容を理解したい。
**それにより**、外国人社員も情報にアクセスできる。

### 機能詳細

| 項目 | 内容 |
|------|------|
| 対象言語 | 英語、ベトナム語、中国語（将来拡張可能） |
| トリガー | 投稿詳細画面の「翻訳する」ボタン |
| 翻訳エンジン | Azure Translator API / LLM |
| キャッシュ | 翻訳結果をDBに保存し、再翻訳を避ける |

### 注意事項

- 現在のMVPは日本語のみ対応のため、この機能は将来的な拡張
- 翻訳精度はビジネスクリティカルではないため、機械翻訳で十分
- 翻訳結果は `post_translations` テーブルに保存

## データベース（AI機能共通）

### posts テーブルへの追加カラム

| カラム | 型 | 説明 |
|--------|-----|------|
| ai_summary | NVARCHAR(500) | AI生成の要約文（NULLable） |

### post_translations テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UNIQUEIDENTIFIER | 主キー |
| post_id | UNIQUEIDENTIFIER | FK → posts |
| language | NVARCHAR(10) | 言語コード（en, vi, zh） |
| translated_body | NVARCHAR(MAX) | 翻訳文 |
| created_at | DATETIME2(7) | 翻訳日時 |

### post_embeddings テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UNIQUEIDENTIFIER | 主キー |
| post_id | UNIQUEIDENTIFIER | FK → posts |
| chunk_index | INT | チャンクインデックス |
| chunk_text | NVARCHAR(1000) | チャンクテキスト |
| embedding | VARBINARY(MAX) | Embeddingベクトル（バイナリ） |
| created_at | DATETIME2(7) | 作成日時 |

## API エンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| `POST` | `/api/v1/ai/suggest` | コンテンツ提案を取得 |
| `POST` | `/api/v1/ai/summarize/:postId` | 投稿の要約を再生成 |
| `POST` | `/api/v1/ai/chat` | チャットボットに質問 |
| `POST` | `/api/v1/ai/translate/:postId` | 投稿を翻訳 |
| `DELETE` | `/api/v1/posts/:postId/ai-summary` | AI要約を削除（管理者） |

## 技術的注意事項

- AI機能はすべてオプショナル（無効化可能）
- LLM呼び出しは非同期キューで処理し、レスポンスタイムに影響を与えない
- オンプレミス環境ではAzure OpenAIへの接続にVPN/ExpressRouteが必要
- ローカルLLM（Ollama等）のサポートも検討し、外部接続不要な構成も可能にする
- AI生成コンテンツには「AI生成」ラベルを付与する
- プロンプトテンプレートは設定ファイルで管理し、チューニング可能にする
