## 2. タイポグラフィ

### フォントファミリー

```css
font-family: 'Noto Sans JP', 'Hiragino Sans', 'Hiragino Kaku Gothic ProN',
             'Yu Gothic', 'Meiryo', sans-serif;
```

Tailwind 設定:
```typescript
// tailwind.config.ts
theme: {
  fontFamily: {
    sans: ['Noto Sans JP', 'Hiragino Sans', 'Yu Gothic', 'Meiryo', 'sans-serif'],
  },
}
```

### フォントサイズ

| 名前 | サイズ | 行間 | 太さ | 用途 |
|------|--------|------|------|------|
| Display | 30px / `text-3xl` | 1.2 | Bold (700) | ページタイトル（大） |
| Heading 1 | 24px / `text-2xl` | 1.3 | Bold (700) | セクションタイトル |
| Heading 2 | 20px / `text-xl` | 1.4 | Semibold (600) | カードタイトル |
| Heading 3 | 18px / `text-lg` | 1.4 | Semibold (600) | サブセクション |
| Body Large | 16px / `text-base` | 1.6 | Regular (400) | 投稿本文 |
| Body | 14px / `text-sm` | 1.5 | Regular (400) | 一般テキスト |
| Caption | 12px / `text-xs` | 1.4 | Regular (400) | 補足情報、日時 |

## 3. スペーシング

### 基本スペーシング

Tailwind のデフォルトスペーシングスケール (4px 単位) を使用。

| トークン | 値 | 用途 |
|---------|------|------|
| `space-1` | 4px | アイコンとテキストの間隔 |
| `space-2` | 8px | コンパクトな内部余白 |
| `space-3` | 12px | ボタン内部余白 |
| `space-4` | 16px | カード内部余白、要素間隔 |
| `space-6` | 24px | セクション間隔 |
| `space-8` | 32px | 大きなセクション間隔 |

### コンポーネント間隔

| コンテキスト | 間隔 |
|-------------|------|
| カード間 | 16px (`gap-4`) |
| カード内セクション間 | 12px (`space-y-3`) |
| フォームフィールド間 | 16px (`space-y-4`) |
| ボタン間 | 8px (`gap-2`) |
| リストアイテム間 | 8px (`space-y-2`) |
| ページコンテンツのパディング | 16px (モバイル) / 24px (デスクトップ) |

## 5. レスポンシブデザイン

### ブレークポイント

| 名前 | 幅 | 説明 |
|------|------|------|
| `sm` | >= 640px | スマートフォン（横） |
| `md` | >= 768px | タブレット |
| `lg` | >= 1024px | デスクトップ（小） |
| `xl` | >= 1280px | デスクトップ（標準） |
| `2xl` | >= 1536px | デスクトップ（大） |

### レイアウト戦略

#### デスクトップ (>= 1024px) — 3カラム

```
┌──────────────────────────────────────────────────────────────────┐
│ Header                                                           │
├──────────┬───────────────────────────────────────┬───────────────┤
│ Sidebar  │          Main Content                  │ Right Panel  │
│ 240px    │          auto                          │ 280px        │
│          │                                        │ (カレンダー等) │
│          │  max-width: 640px (コンテンツ幅)        │              │
│          │                                        │              │
└──────────┴───────────────────────────────────────┴───────────────┘
```

#### タブレット (768px - 1023px) — 2カラム

```
┌────────────────────────────────────────────────┐
│ Header                                          │
├───────────┬────────────────────────────────────┤
│ Sidebar   │       Main Content                  │
│ 200px     │       auto                          │
│ (折りたたみ│                                     │
│  可能)    │                                     │
└───────────┴────────────────────────────────────┘
```

#### モバイル (< 768px) — 1カラム + ボトムナビ

```
┌────────────────────┐
│ Header (簡易)       │
├────────────────────┤
│                    │
│   Main Content     │
│   (フル幅)          │
│                    │
├────────────────────┤
│ Bottom Navigation  │
└────────────────────┘
```

```

```

**スタイル:**
- テキスト: text-sm, text-gray-500, font-medium
- 区切り線: gray-200
- レイアウト: flex items-center, 左右に hr

**モバイル:** ボトムナビの「+」ボタンでフルスクリーンモーダル

### 6.4 Navigation（サイドバーナビゲーション）

```
┌─────────────────┐
│ 🏠 タイムライン   │ ← アクティブ: bg-primary-50, text-primary-700
│ 📢 お知らせ  (3) │ ← 未読バッジ
```

**スタイル:**
- 幅: 240px (デスクトップ)
- アイテム高さ: 40px
- アクティブ: bg-primary-50, text-primary-700, border-l-2 border-primary-600
- ホバー: bg-gray-50

```
  🔔
 (5)  ← 赤丸バッジ（未読件数）
```

**バッジスタイル:**
- 背景: bg-red-500
- テキスト: text-white, text-xs, font-bold
- サイズ: min-w-5 h-5
- 位置: absolute -top-1 -right-1
- 99件超えの場合: "99+"

## 7. インタラクションパターン

### 7.1 ローディング状態

**スケルトンローディング:**
投稿カード、お知らせカード等のプレースホルダーをスケルトンで表示。

```
┌─────────────────────────────────────────────┐
│ ┌──┐  ████████████  ██████                  │
│ │░░│  ██████████                             │
│ └──┘                                        │
│ ████████████████████████████████████         │
│ ████████████████████████████                 │
│ ███████████████████                         │
│                                             │
│ ██ ██  ██ ██                                │
└─────────────────────────────────────────────┘
```

スケルトンアニメーション: `animate-pulse` (Tailwind)

**ボタンローディング:**
- ボタン内にスピナーを表示
- ボタンを `disabled` にして二重送信を防止

### 7.2 空状態

コンテンツがない場合のプレースホルダー表示。

```
┌─────────────────────────────────────────────┐
│                                             │
│              📝                             │
│                                             │
│         投稿がありません                      │
│   まだ誰も投稿していません。                   │
│   最初の投稿を作成しましょう！                 │
│                                             │
│          [ 投稿を作成する ]                   │
│                                             │
└─────────────────────────────────────────────┘
```

**画面別の空状態メッセージ:**

| 画面 | アイコン | タイトル | 説明 |
|------|---------|---------|------|
| タイムライン（日付指定） | 📝 | この日の投稿はありません | — |
| お知らせ | 📢 | お知らせはありません | 新しいお知らせが投稿されるとここに表示されます |
| アンケート | 📋 | アンケートはありません | 新しいアンケートが作成されるとここに表示されます |
| 通知 | 🔔 | 通知はありません | 新しい通知はここに表示されます |
| コメント | 💬 | コメントはありません | 最初のコメントを投稿しましょう |

### 7.3 トースト通知

[Sonner](https://sonner.emilkowal.dev/) を使用。

| 種類 | 色 | 表示時間 | 例 |
|------|-----|---------|-----|
| Success | 緑 | 3秒 | 「投稿を作成しました」 |
| Error | 赤 | 5秒 | 「投稿の作成に失敗しました」 |
| Info | 青 | 3秒 | 「田中さんがいいねしました」 |
| Warning | 黄 | 4秒 | 「アンケートの締切が近づいています」 |

位置: 画面右上 (`top-right`)

### 7.4 リアルタイム更新

| イベント | UI更新 |
|---------|--------|
| 新着いいね | 投稿カードのいいね数がアニメーション付きで更新 |
| 新着コメント | 投稿カードのコメント数が更新。コメント展開中は新コメントが追加 |
| 新着通知 | 通知ベルのバッジ数が更新 + トースト表示 |
| 新着お知らせ | トースト表示 + お知らせバナー更新 |

### 7.5 確認ダイアログ

破壊的操作（削除等）の前に確認ダイアログを表示。

```
┌─────────────────────────────────┐
│         投稿を削除しますか？      │
│                                 │
│ この操作は取り消せません。       │
│                                 │
│       [ キャンセル ] [ 削除 ]    │
│                                 │
└─────────────────────────────────┘
```

- 「削除」ボタン: bg-red-600, text-white
- 「キャンセル」ボタン: bg-white, border, text-gray-700

## 8. アクセシビリティ

### 基本方針

- WCAG 2.1 Level AA 準拠を目指す
- キーボード操作に完全対応
- スクリーンリーダー対応（適切な ARIA ラベル）

### 実装チェックリスト

| 項目 | 実装 |
|------|------|
| カラーコントラスト比 4.5:1 以上 | Tailwind のカラーで担保 |
| フォーカスインジケーター | `focus-visible:ring-2 focus-visible:ring-primary-500` |
| 画像の alt 属性 | すべての画像に日本語 alt テキスト |
| フォームラベル | すべての入力フィールドに label 要素 |
| エラーメッセージ | `aria-describedby` でフィールドに紐付け |
| ページタイトル | 各ページに適切な `<title>` |
| 見出しの階層 | h1 → h2 → h3 の順序を遵守 |
| ナビゲーション | `<nav>` 要素 + `aria-label` |
| モーダル | フォーカストラップ + `aria-modal` |
| ローディング状態 | `aria-busy` + `aria-live="polite"` |

## 9. Tailwind 設定

```typescript
// tailwind.config.ts
import type { Config } from 'tailwindcss';

const config: Config = {
  darkMode: 'class',
  content: ['./src/**/*.{ts,tsx}'],
  theme: {
    extend: {
      colors: {
        primary: {
          50: '#eff6ff',
          100: '#dbeafe',
          200: '#bfdbfe',
          300: '#93c5fd',
          400: '#60a5fa',
          500: '#3b82f6',
          600: '#2563eb',
          700: '#1d4ed8',
          800: '#1e3a8a',
          900: '#1e3a5c',
        },
      },
      fontFamily: {
        sans: [
          'Noto Sans JP',
          'Hiragino Sans',
          'Hiragino Kaku Gothic ProN',
          'Yu Gothic',
          'Meiryo',
          'sans-serif',
        ],
      },
      maxWidth: {
        content: '640px',
      },
    },
  },
  plugins: [require('tailwindcss-animate')],
};

export default config;
```
