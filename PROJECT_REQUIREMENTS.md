# プロジェクト要件書 - コープネット

> 社内SNSシステム

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | コープネット（CoopNet） |
| 種別 | 社内ソーシャルネットワーク |
| 対象ユーザー数 | 100〜150名 |
| 対応言語 | 日本語のみ |
| 運用環境 | オンプレミスサーバー + VPN |
| 開発体制 | 1名 + AIアシスタント |

### 目的

社員同士のコミュニケーション活性化と情報共有の促進を目的とした社内SNSシステム。毎日の業務報告や日常の投稿を通じて、部署間の壁を越えた交流を実現する。

---

## 2. ビジネスルール

### 投稿ルール

| ルール | 詳細 |
|--------|------|
| 毎日投稿 | 全社員は1日1回以上の投稿が求められる |
| バックデート | 投稿日を過去の日付に設定可能（投稿忘れ対応） |
| 単一タイムライン | 全社員が同じタイムラインを閲覧する |
| 投稿形式 | テキスト + 画像添付（複数可） |

### コンテンツルール

| ルール | 詳細 |
|--------|------|
| いいね | 1投稿につき1ユーザー1回 |
| コメント | 投稿へのコメント（ネストなし） |
| お知らせ | 管理者のみ作成可能、全社員に表示 |
| アンケート | 管理者が作成、全社員が回答 |

## 6. 非機能要件

### パフォーマンス

| 指標 | 目標値 |
|------|--------|
| ページ読み込み時間 | 2秒以内 |
| API レスポンスタイム（95パーセンタイル） | 500ms以内 |
| 同時接続ユーザー数 | 150名 |
| タイムライン1ページあたり | 20投稿 |

### レスポンシブデザイン

| デバイス | 対応 |
|----------|------|
| デスクトップ（1280px以上） | フル機能 |
| タブレット（768px〜1279px） | フル機能（レイアウト調整） |
| スマートフォン（〜767px） | 主要機能（投稿閲覧・作成・いいね・コメント） |

### リアルタイム

| 機能 | 方式 |
|------|------|
| 通知 | WebSocket (Socket.IO) |
| タイムライン更新 | WebSocket（新着投稿の通知バナー） |
| いいね数更新 | WebSocket |

### セキュリティ

| 項目 | 対応 |
|------|------|
| 認証 | JWT（アクセストークン15分、リフレッシュトークン7日） |
| パスワード | bcryptハッシュ、最低8文字 |
| CORS | フロントエンドのドメインのみ許可 |
| ファイルアップロード | ファイル種別・サイズ制限、ウイルススキャン検討 |
| SQLインジェクション | TypeORM パラメータ化クエリ |
| XSS | React の自動エスケープ + サニタイズ |

## 8. データ移行

### 既存データ

現在は Excel ファイルで日報・投稿を管理している。MVP リリース時に以下の移行を行う:

| データ | 移行元 | 優先度 |
|--------|--------|--------|
| ユーザーマスタ | Excel / Active Directory | 高 |
| 過去の投稿（直近3ヶ月） | Excel | 中 |
| 画像ファイル | 共有フォルダ | 低 |

### 移行方法

- バックエンドにCSVインポートAPIを作成
- 管理者画面からCSVファイルをアップロードして一括インポート
- インポート前にバリデーション、エラーログ出力
